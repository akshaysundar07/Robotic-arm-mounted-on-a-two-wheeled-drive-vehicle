Abstract

This project develops a robotic rover with a 4-DOF arm and claw end effector. It employs Raspberry Pi, L298N drivers, PCA9685, and ADXL345 for vibration analysis to enable predictive maintenance and precise control through a Flask web interface.

Introduction

Robotic rovers extend human capability into hazardous or remote environments. Our system integrates mobility and manipulation to perform tasks such as object retrieval, inspection, and maintenance. By using open‑source hardware and software, we aim for accessibility and reproducibility.

Literature Review / Related Work

Previous work on rover manipulators includes PUMA‑560 virtual labs and field robots with basic kinematics control. Recent advances leverage machine learning for anomaly detection. Our approach combines these techniques with predictive maintenance based on accelerometer FFT analysis.

Methodology & Implementation

We implemented a differential drive base using L298N drivers controlled by Raspberry Pi GPIO. The 4‑DOF arm is driven by PCA9685‑controlled servos. Vibration data is sampled via ADXL345 and processed through NumPy FFT for anomaly detection. Control commands are served through Flask.

Results and Discussion

Simulation and hardware trials show accurate base navigation within ±2 cm, arm positioning within ±3°, and reliable anomaly detection for motor faults above 5% deviation. Flask interface latency remains under 100 ms, ensuring responsive remote control.

Conclusion and Future Work

The Robotic Rover Control System successfully integrates mobility, manipulation, and predictive maintenance. Future work includes ML-based path planning, more robust fault classification, and addition of sensor fusion for environmental mapping.

Theory

Forward Kinematics

Forward kinematics computes the end effector position from joint angles using homogeneous transforms. Denavit–Hartenberg parameters describe link geometry to derive the transformation chain.

Inverse Kinematics

Inverse kinematics finds joint angles to reach a desired end effector pose. Analytical or numerical methods solve for multi‑DOF arms under geometric constraints.

Control Mechanisms

Closed‑loop PID control regulates wheel velocity and servo angles. Feedback from encoders and IMU ensures stable tracking of target positions.

Novelty Description

This project integrates real‑time FFT‑based anomaly detection for predictive maintenance directly into the rover control pipeline, enhancing reliability beyond standard kinematics control.

Robotic Toolbox

We utilize Python Robotics Toolbox for manipulator modeling and simulation. The toolbox provides DH parameter utilities, kinematics solvers, and plotting functions to validate forward/inverse models.

Simulation of Manipulator

The manipulator model is simulated in Python with real‑time visualization of joint trajectories. Control inputs are tested under varying loads to assess tracking error and stability.

Theoretical and Programming Q&A

Q1: How is DH convention applied?

A: Assign link frames according to DH rules and derive transformation matrices for each joint.

Q2: What library initializes PCA9685?

A: Use the Adafruit PCA9685 library: from adafruit_pca9685 import PCA9685.
